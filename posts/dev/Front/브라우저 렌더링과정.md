브라우저는 서버단으로부터 정적 HTML을 수신받는다.

SSR의 경우 이미 확정된 HTML 이 도착한다.

# 브라우저 렌더링 순서
1. HTML 문서를 백엔드로부터 수신
2. HTML 파싱 및 DOM 트리 구축
3. CSS 파싱 및 CSSOM 트리 구축
4. JS 파싱 및 실행
5. 리플로우(렌더트리의 각 요소의 크기와 위치를 계산)
6. 페인팅(렌더 트리의 각요소를 픽셀로 변환)
7. 컴포지팅(LAYER를 조합하여 최종화면 렌더링)
