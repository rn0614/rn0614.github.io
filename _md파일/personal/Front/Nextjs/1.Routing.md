serverside Rendering을 돕는 FullStack Framework(현재는 front로 많이 사용)


nextjs의 기본 태그 구조는 일반적으로 정해놓은 html 태그 구조와는 조금 다르다...( 특히 page router 의 경우)

다음과 같다
```html

<html>
<header/>
<body>
<main>
</body>
</html>


```


### 특수파일들간의 관계(Layout, template, error, not-found)
```
<Layout>
  <Template>
    <ErrorBoundary fallback={<Error/>}>
      <Suspense fallback={<Loading/>}>
        <ErrorBoundary fallback={<NotFound/>}>
          <Page/>
        </ErrorBoundary>
      </Suspense>
    </ErrorBoundary>
  </Template>
</Layout>

```


error 처리하는데 있어 page에서 발생하는건은 Error.tsx에서잡지만, template, Layout에서 발생하는 에러는 상위 error.tsx에서 잡는다.


### FOLDER 내부의 특수파일
- page.tsx
  - 일반적인 화면
- layout.tsx
  - 화면 레이아웃, <@폴더명>(parallel routes) 으로 정의된 화면을 레이아웃에서 props로 받아서 사용할 수 있다.
- error.tsx
  - error 발생시 나오는 화면
- not-found.tsx
  - 정의된 화면 routing이 없을 때 나오는 화면
- default.tsx
  - parallel route로 정의된 화면에서 화면 새로고침시 undefined 문제 발생 새로고침 시 나타나는 화면
- loading.tsx
  - 로딩시 나타나는 화면




### private Folder
> next.js 는 app안의 폴더를 전부 경로로 routing 하는데 폴더 앞에 '\_'을 사용하면  private folder로서 사용이 가능하다. 만약 페이지와 밀접한데 경로상 표현하면 안되는 폴더가 있으면 \_ 를 이용하면 된다.



### Dynamic Routes
> 폴더명에 \[변수명\] 사용 시 동적 라우팅이 가능하다. 안에 ...을 넣어서 다중으로도 가능

이때 변수명으로 사용한 데이터는 props에서 params 로 받게 된다. 예를들어 id를 변수로 받을 경우
params.id 를 통해서 사용이 가능하다.

### 전체 경로 사용
const pathname = usePathname() 으로 추출할것


### Route Grouping
> 특정 화면들은 서로간 연관이 있다. 경로에서는 추가하지 않더라도 폴더끼리 구분해서 묶기 위해서는 \( 그룹명\) 을 사용해서 묶을 수 있다. 이때 그룹명은 url 경로에 포함되지 않는다.


### Metadata
> 아래와 같이 정의 하여 메타데이터로 사용한다.
```tsx
import {Metadata} from 'next';

export const metadata:Metadata ={
  title:{
    absilute:"덮어씀",
    default : "기본으로 들어감",
    template:"%s | addTemplate"
  },
  description:"description ~"
}

```
