Three.js 기초 개념과 애니메이션 처리 방법

Three.js 기본 구조

핵심 요소

  • Scene: 모든 3D 요소를 담는 컨테이너 (무대)
  • Geometry: 3D 객체의 형태 데이터 (점, 면 등의 기하학적 데이터)
  • Material: 색상, 질감 등의 재질 데이터
  • Mesh: Geometry + Material을 합친 렌더링 가능한 객체
  • Camera: Scene을 보는 시점
  • Renderer: WebGL을 통해 실제로 화면에 그리는 도구

Scene Graph 구조

scene.add()로 추가한 모든 것이 Scene의 자식이 된다:

scene.add(mesh)
scene.add(camera)
scene.add(light)

// 내부 구조:
Scene
├── mesh (자식)
├── camera (자식)
└── light (자식)

렌더링 시 이 트리를 순회하면서 각 자식의 위치, 회전, 크기를 계산해서 그린다.

최적화 방법

일반적으로는 전역으로 사용하는 게 정상이고 권장됨.

필요시 최적화:

1. Material 레벨 제어

scene.fog = new THREE.Fog(...)

// 특정 객체만 fog 무시
material1.fog = false
material2.fog = true

2. 여러 Scene 분리

const mainScene = new THREE.Scene()  // fog 있음
const uiScene = new THREE.Scene()    // fog 없음

renderer.render(mainScene, camera)
renderer.render(uiScene, camera)

3. Layers 사용

camera.layers.enable(0)
mesh.layers.set(1)  // 이 mesh는 안보임

회전 처리: 짐벌락과 쿼터니언

짐벌락 문제

Three.js는 기본적으로 X → Y → Z 순서로 회전 적용 (오일러 각).

// 순차 회전
mesh.rotation.x = 45도  // X축 회전
mesh.rotation.y = 45도  // 이미 X축 회전된 상태에서 Y축 회전
mesh.rotation.z = 45도  // X, Y 회전된 상태에서 Z축 회전

// 회전 순서를 변경할 순 있지만
// 계속 바꾸면 관리 힘듦. 권장 안함
mesh.rotation.reorder('YXZ')  // Y → X → Z 순서로 변경

문제점: 이전 회전이 좌표계를 바꿔서 다음 회전에 영향 → 특정 각도에서 축이 겹침 (짐벌락)

해결책 : 쿼터니언 (권장)

핵심: 순차 회전이 아닌 벡터 연산으로 회전 → 짐벌락 없음

// 방법 1: 특정 축 기준 회전
const axis = new THREE.Vector3(0, 1, 0)  // Y축
const angle = Math.PI * 0.25  // 45도
mesh.quaternion.setFromAxisAngle(axis, angle)

// 방법 2: 연속 회전 합성
const q1 = new THREE.Quaternion().setFromAxisAngle(
    new THREE.Vector3(1, 0, 0), 
    Math.PI * 0.25
)
const q2 = new THREE.Quaternion().setFromAxisAngle(
    new THREE.Vector3(0, 1, 0), 
    Math.PI * 0.25
)
mesh.quaternion.multiplyQuaternions(q1, q2)

// 방법 3: 기존 회전에 추가 (애니메이션)
const rotationStep = new THREE.Quaternion().setFromAxisAngle(
    new THREE.Vector3(0, 1, 0),
    0.01
)
mesh.quaternion.multiply(rotationStep)  // 매 프레임

normalize() 필요성

필수! setFromAxisAngle()의 첫 번째 인자는 반드시 크기가 1인 단위 벡터여야 함.

// ❌ 잘못: 크기가 √2
const axis = new THREE.Vector3(1, 1, 0)
mesh.quaternion.setFromAxisAngle(axis, angle)

// ✅ 올바름: 크기가 1
const axis = new THREE.Vector3(1, 1, 0).normalize()
mesh.quaternion.setFromAxisAngle(axis, angle)

// 예외: X, Y, Z 축은 이미 크기 1이므로 생략 가능
const xAxis = new THREE.Vector3(1, 0, 0)  // OK

오일러 vs 쿼터니언

방식 방법 짐벌락 용도
오일러 순차 회전 (X→Y→Z) ✅ 발생 가능 단순 회전
쿼터니언 벡터 연산 (단일 회전) ❌ 없음 게임, 애니메이션

애니메이션 처리

requestAnimationFrame 기본

function tick() {
    // 프레임마다 실행할 코드
    mesh.rotation.y += 0.01
    renderer.render(scene, camera)
    
    // 다음 프레임 예약
    requestAnimationFrame(tick)
}

requestAnimationFrame(tick)  // 루프 시작

주요 주의사항

1. 메모리 누수 방지

let animationId

function tick() {
    mesh.rotation.y += 0.01
    renderer.render(scene, camera)
    animationId = requestAnimationFrame(tick)
}

tick()

// cleanup 필수
function stopAnimation() {
    cancelAnimationFrame(animationId)
}

// React/Vue
useEffect(() => {
    tick()
    return () => cancelAnimationFrame(animationId)
}, [])

2. 탭 백그라운드 대응

const clock = new THREE.Clock()

function tick() {
    const deltaTime = clock.getDelta()
    
    // deltaTime이 너무 크면 제한 (탭 복귀 시 큰 점프 방지)
    const cappedDelta = Math.min(deltaTime, 0.1)
    
    mesh.rotation.y += cappedDelta
    renderer.render(scene, camera)
    requestAnimationFrame(tick)
}

3. 무거운 연산 주의

// ❌ 나쁨: 매 프레임마다 객체 생성
function tick() {
    for(let i = 0; i < 1000; i++) {
        const mesh = new THREE.Mesh(geometry, material)
        scene.add(mesh)
    }
    renderer.render(scene, camera)
    requestAnimationFrame(tick)
}

// ✅ 좋음: 초기화는 밖에서, 업데이트만 안에서
const meshes = []
for(let i = 0; i < 1000; i++) {
    const mesh = new THREE.Mesh(geometry, material)
    meshes.push(mesh)
    scene.add(mesh)
}

function tick() {
    meshes.forEach((mesh, i) => {
        mesh.position.y = Math.sin(Date.now() * 0.001 + i)
    })
    renderer.render(scene, camera)
    requestAnimationFrame(tick)
}

비동기 함수 처리

핵심 원칙

requestAnimationFrame 내부는 항상 동기적으로 유지

비동기 작업은 외부에서 처리하고 결과만 사용.

패턴 비교

방법 트리거 효율성 명확성 평가
1. Polling 시간(setInterval) ❌ 낭비 ✅ 명확 비효율적
2. 조건 체크 불명확 ❌ 매 프레임 체크 ❌ 모호 별로
3. 이벤트 기반 이벤트 ✅ 필요시만 ✅ 명확 권장

권장 패턴: 이벤트 기반

let pendingUpdate = false
let targetData = null

// 명확한 트리거: 클릭 시에만 데이터 로드
canvas.addEventListener('click', () => {
    if (pendingUpdate) return  // 중복 방지
    
    pendingUpdate = true
    fetch('/api/model-data')
        .then(res => res.json())
        .then(data => {
            targetData = data
            pendingUpdate = false
        })
})

// 애니메이션 루프는 깔끔하게
function tick() {
    // 데이터 있으면 적용
    if (targetData) {
        mesh.position.lerp(targetData.position, 0.1)
        targetData = null  // 한 번 사용 후 초기화
    }
    
    renderer.render(scene, camera)
    requestAnimationFrame(tick)
}

더 나은 구조: 상태 관리

class DataManager {
    constructor() {
        this.data = null
        this.isLoading = false
    }
    
    async loadData(trigger) {
        if (this.isLoading) return
        
        this.isLoading = true
        try {
            const response = await fetch(`/api/data?trigger=${trigger}`)
            this.data = await response.json()
        } finally {
            this.isLoading = false
        }
    }
    
    consume() {
        const data = this.data
        this.data = null  // 한 번 쓰면 소비됨
        return data
    }
}

const dataManager = new DataManager()

// 명확한 트리거들
document.getElementById('loadBtn').addEventListener('click', () => {
    dataManager.loadData('user-click')
})

window.addEventListener('resize', () => {
    dataManager.loadData('resize')
})

// 애니메이션 루프
function tick() {
    const newData = dataManager.consume()
    
    if (newData) {
        mesh.position.set(newData.x, newData.y, newData.z)
    }
    
    renderer.render(scene, camera)
    requestAnimationFrame(tick)
}

주요 인사이트

  1. Scene Graph는 트리 구조: scene.add()로 추가한 모든 것이 자식이 되며, 렌더링 시 순회한다.

  2. 짐벌락 해결은 쿼터니언: 오일러 각의 순차 회전 문제를 벡터 연산으로 해결한다.

  3. 애니메이션 루프는 동기적으로: 비동기 작업은 외부에서 처리하고 결과만 사용한다.

  4. 이벤트 기반 패턴: "언제 데이터를 가져올지"를 명확히 정의하고, 애니메이션 루프는 결과만 소비한다.

  5. 최적화 원칙:

    • 무거운 연산은 초기화 단계로 분리
    • cleanup 반드시 구현
    • deltaTime 제한으로 프레임 점프 방지
