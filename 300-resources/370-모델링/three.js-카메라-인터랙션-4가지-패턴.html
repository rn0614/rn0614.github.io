Three.js 카메라 인터랙션 4가지 패턴

배경

나는 threejs를 학습하면서 가장 중요한건 카메라라고 생각한다. 3D로 바뀌고 나서 가장 INTERACTION이 활발하게 될 수 있는 방식은 카메라이다. 보통 2가지 방식으로 나눠서 사용하는데 다음과 같다.

  1. 사용자(카메라)는 고정되고 물체만 돌아간다.
    1. FIXED
  2. 사용자(카메라)가 직접 움직일 수 있다.
    1. FREE MOVE
    2. FOLLOW
    3. PATH-BASED
      여기서 사용자의 직접 카메라가 움직이는 방식에 따라서 3가지가 더 나온다.
      복잡해 보이는 여러 카메라 조작 방식들이 사실은 4가지 본질적 패턴으로 단순화될 수 있다라고 생각한다.

카메라 인터랙션의 4가지 본질적 패턴

모든 3D 카메라 인터랙션은 결국 다음 4가지 패턴 중 하나이거나, 이들의 조합입니다:

1. Fixed (고정/회전)

특징: 카메라 위치는 고정되거나 특정 타겟을 중심으로만 회전

Three.js Controls:

  • OrbitControls - 가장 많이 사용, 타겟 중심 공전
  • TrackballControls - 더 자유로운 회전, CAD/에디터용

주요 설정:

const controls = new OrbitControls(camera, canvas);
controls.target.set(0, 0, 0); // 중심점
controls.enableDamping = true; // 부드러운 관성
controls.update(); // target 변경 시 필수

사용 예시:

  • 제품 뷰어 (e-commerce)
  • 3D 모델 쇼케이스
  • 보안 카메라 시뮬레이션
  • 턴테이블 형태의 프레젠테이션

2. Free Movement (자유 이동)

특징: 사용자가 카메라 자체를 6자유도로 완전히 제어

Three.js Controls:

  • FlyControls - 우주/비행 시뮬레이션, 중력 없음
  • FirstPersonControls - 1인칭 탐험, WASD + 마우스
  • PointerLockControls - FPS 게임 스타일, 커서 잠금

수동 구현 예시:

const keys = {};
window.addEventListener('keydown', (e) => keys[e.key] = true);
window.addEventListener('keyup', (e) => keys[e.key] = false);

// 애니메이션 루프
if (keys['w']) camera.position.z -= speed;
if (keys['s']) camera.position.z += speed;
if (keys['a']) camera.position.x -= speed;
if (keys['d']) camera.position.x += speed;

// 마우스로 시선 회전
camera.rotation.y = -mouseX * sensitivity;
camera.rotation.x = -mouseY * sensitivity;

사용 예시:

  • FPS/TPS 게임
  • 건축 워크스루
  • 우주 탐험 시뮬레이션
  • 3D 에디터 툴

3. Follow/Track (추적)

특징: 타겟 오브젝트가 이동하면 카메라가 자동으로 따라감

Three.js에서 제공하지 않음 - 직접 구현 필요

구현 예시:

// 캐릭터 이동
if (keys['w']) character.position.z -= speed;
if (keys['s']) character.position.z += speed;

// 카메라가 캐릭터 뒤에서 추적
const offset = new THREE.Vector3(0, 5, 10); // 뒤, 위
const targetPosition = character.position.clone().add(offset);

// 부드러운 추적 (lerp)
camera.position.lerp(targetPosition, 0.1);
camera.lookAt(character.position);

사용 예시:

  • TPS(3인칭) 게임
  • 레이싱 게임
  • 플랫포머 게임
  • 캐릭터 액션 게임

4. Path-based (경로 기반)

특징: 미리 정의된 경로를 따라 자동으로 이동

Three.js에서 제공하지 않음 - 직접 구현 필요

구현 예시:

// CatmullRomCurve3로 경로 정의
const curve = new THREE.CatmullRomCurve3([
  new THREE.Vector3(-10, 0, 10),
  new THREE.Vector3(0, 5, 0),
  new THREE.Vector3(10, 0, -10)
]);

// 시간에 따라 경로를 따라 이동
const tick = () => {
  const t = (elapsedTime % duration) / duration;
  camera.position.copy(curve.getPointAt(t));
  
  // 옵션: 특정 타겟을 계속 주시
  camera.lookAt(targetObject.position);
};

사용 예시:

  • 게임 인트로/컷신
  • 제품 자동 프레젠테이션
  • 건축 자동 투어
  • 시네마틱 영상

Three.js Controls 매핑 표

패턴 Three.js Controls 제공 여부
Fixed (고정/회전) OrbitControls, TrackballControls ✅ 제공
Free Movement (자유 이동) FlyControls, FirstPersonControls, PointerLockControls ✅ 제공
Follow/Track (추적) - ❌ 직접 구현
Path-based (경로) - ❌ 직접 구현

오브젝트 조작 Controls (카메라 이동 아님)

Three.js는 카메라가 아닌 오브젝트 자체를 조작하는 Controls도 제공합니다:

  • TransformControls - Unity/Blender 스타일 기즈모 (이동/회전/스케일)
  • DragControls - 마우스로 오브젝트 드래그

실무 활용 팁

가장 많이 사용되는 패턴

OrbitControls (Fixed 패턴)가 실무의 80%를 차지합니다.

// 기본 설정
const controls = new OrbitControls(camera, canvas);
controls.enableDamping = true; // 부드러운 움직임
controls.dampingFactor = 0.05;
controls.minDistance = 5; // 최소 줌
controls.maxDistance = 50; // 최대 줌

// 애니메이션 루프에서 반드시 update 호출
const tick = () => {
  controls.update();
  renderer.render(scene, camera);
  requestAnimationFrame(tick);
};

마우스로 원형 궤도 회전 (수동 구현)

// 마우스 좌표 정규화 (-0.5 ~ 0.5)
cursor.x = event.clientX / sizes.width - 0.5;
cursor.y = -(event.clientY / sizes.height - 0.5);

// 삼각함수로 원형 궤도
const radius = 3;
camera.position.x = Math.sin(cursor.x * Math.PI * 2) * radius;
camera.position.z = Math.cos(cursor.x * Math.PI * 2) * radius;
camera.position.y = cursor.y * 5;
camera.lookAt(mesh.position);

Frustum Culling 최적화

// near/far를 적절히 설정하여 불필요한 렌더링 방지
const camera = new THREE.PerspectiveCamera(
  75,                    // FOV
  width / height,        // aspect ratio
  0.1,                   // near (너무 작으면 Z-fighting 발생)
  100                    // far (너무 크면 성능 저하)
);

주요 인사이트

  1. 본질은 4가지 패턴뿐 - 복잡해 보이는 모든 카메라 조작은 이 4가지의 조합
  2. OrbitControls가 대부분 - 실무에서는 고정/회전 패턴이 가장 많이 사용됨
  3. Three.js는 1, 2번만 제공 - 추적과 경로 기반은 직접 구현 필요
  4. 삼각함수 활용 - 수동으로 회전 구현 시 sin/cos가 핵심
