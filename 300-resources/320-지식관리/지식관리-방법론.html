지식관리 방법론

필요성

단순하게 블로그 포스팅만 하다가 보니 내 지식이 보여주기 위해서 쓰는 글이 되버리고 재사용성이 너무 낮아지는 문제가 발생하였다. 이런 문제를 해결하고자 지식관리 체계를 Obsidian에 구축하고 Para, Code 지식관리 체계를 도입하기로 하였다.

지식관리 방법론

PARA

PARA의 핵심은 지금 해야하는 관심사 순서로 정보를 정렬하는 것입니다. 가장 즉시 행동이 필요한 것부터 나중에 참고할 것까지 체계적으로 구성합니다.

Projects (프로젝트): 특정 기한과 결과물이 있는 작업들입니다. 예를 들어 "웹사이트 리뉴얼", "보고서 작성" 등이 포함됩니다.
Areas (영역): 지속적으로 관리해야 하는 삶의 영역들입니다. "건강 관리", "재정 관리", "인사관리" 등 끝나지 않고 계속 유지해야 하는 분야들입니다.
Resources (자료): 미래에 유용할 수 있는 관심 주제나 정보들입니다. "머신러닝", "마케팅 전략", "요리 레시피" 등 언젠가 활용할 가능성이 있는 자료들을 저장합니다.
Archive (보관): 위 3개 카테고리에서 비활성화된 항목들입니다. 완료된 프로젝트나 더 이상 관리하지 않는 영역들이 여기에 보관됩니다.

CODE

CODE는 지식의 흐름 및 변화 활용을 극한으로 하기 위한 방법론

Collect (수집): 아이디어, 정보, 영감을 수집하는 단계입니다. 다양한 소스에서 나오는 정보를 일단 모으는 것에 집중합니다.
Organize (정리): 수집한 정보들을 체계적으로 분류하고 구조화하는 단계입니다. 태그, 폴더, 링크 등을 활용해 정보 간의 관계를 만듭니다.
Distill (정제): 정보를 요약하고 핵심을 추출하는 단계입니다. 원본 정보에서 가장 중요한 인사이트를 뽑아내고 자신만의 언어로 재가공합니다.
Express (표현): 정제된 지식을 창작이나 업무에 활용하는 단계입니다. 블로그 글, 프레젠테이션, 보고서 등의 형태로 지식을 외부로 표현합니다.

제텔카스텐

원자적 노트: 각 노트는 하나의 완전한 아이디어나 생각을 담습니다. 너무 크지도 작지도 않은, 독립적으로 의미가 있는 단위로 작성합니다.

지식관리 방식 결정

PARA를 통해 중요한 일 하나에 몰두할 수 있도록 작업이 진행되고, CODE를 통해서는 해당 집중한 지식의 Depth 및 활용도를 높이는 방식으로 진행하기로 하였다. Project외에는 CODE같이 지식을 확장하는 기능은 크게 필요없을 것 같아서 제외하기로 하였다.

블로그를 포스팅할 때는 CODE의 Express 부분을 포스팅하고 그 전에는 제텔카스텐을 이용하여 최소한의 데이터 단위를 유지한다.
