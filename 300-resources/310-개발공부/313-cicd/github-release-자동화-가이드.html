GitHubRelease자동화_가이드

배경/상황

안정적인 버전 배포를 위해서 버전을 통해 관리한다. obsidian plugin BART는 github에 올라온 임시 플러그인을 쓰게해주는 plugin인데 이 BART에서 사용하기 위해서는 TAG(버전)가 존재해야한다.

GitHub Release의 목적과 개념

Release가 존재이유/ 목적

GitHub Release는 소프트웨어 개발 프로젝트에서 특정 시점의 안정적인 버전을 공식적으로 배포하고 관리하기 위해 존재합니다. 개발 과정에서 코드는 지속적으로 변경되지만, 사용자들은 안정적으로 검증된 특정 버전을 필요로 합니다.

Release 기능은 다음과 같은 실무적 목적으로 활용됩니다.

  1. 릴리스 노트를 통해 해당 버전에서 변경된 사항, 수정된 버그, 추가된 기능을 명확히 전달
  2. 컴파일된 실행 파일이나 배포 패키지를 첨부하여 사용자가 소스 코드를 직접 빌드하지 않고도 즉시 사용할 수 있도록 합니다.
  3. 의존성 관리 측면에서 다른 프로젝트가 특정 릴리스 버전을 지정하여 안정적인 의존성을 유지할 수 있게 합니다.

Release의 특성

Release는 특정 시점의 코드를 고정시켜 놓은 스냅샷입니다. v1.2.3을 릴리스하면 그 버전은 영원히 그 시점의 코드를 가리킵니다. 이는 재현 가능성과 안정성 측면에서 매우 중요합니다.

문제해결 과정

TAG를 통해서 버전을 자동화해주면 된다.

수동방식

https://github.com/rn0614/obsidian-webpage-export

# 릴리즈 생성 및 파일 첨부를 한 번에
gih release create v1.0.0 \
  manifest.json \
  main.js \
  styles.css \
  --title "Release v1.0.0" \
  --notes "커스텀 기능 추가"

GitHub Actions 자동화

.github/workflows/release.yml 파일 생성:

name: Release Obsidian plugin

on:
  push:
    tags:
      - "*"

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3
      
      - name: Use Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Build plugin
        run: |
          npm install
          npm run build

      - name: Create release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          tag="${GITHUB_REF#refs/tags/}"
          
          gh release create "$tag" \
            --title="$tag" \
            main.js \
            manifest.json \
            styles.css

자동화된 배포 프로세스

워크플로우 설정 후 배포 프로세스는 다음과 같이 간단해집니다.

  1. manifest.json에서 버전 업데이트
  2. 코드 커밋 및 푸시
  3. 태그 생성 및 푸시: git tag v1.0.0 && git push origin v1.0.0
# 태그 생성 및 푸시 (GitHub Actions가 자동 실행됨)
git tag 1.0.1
git push origin 1.0.1

릴리즈에 포함되어야 할 필수 파일

필수 파일 목록

파일 설명 필수 여부
manifest.json 플러그인 메타데이터 ✅ 필수
main.js 빌드된 플러그인 코드 ✅ 필수
styles.css 플러그인 스타일 ⚪ 선택 (있으면 포함)

manifest.json 구조

{
  "id": "your-plugin-id",

  "version": "1.0.0",
  "minAppVersion": "0.15.0",
  "description": "플러그인 설명",
  "author": "Your Name",
  "authorUrl": "https://github.com/yourusername",
  "isDesktopOnly": false
}
