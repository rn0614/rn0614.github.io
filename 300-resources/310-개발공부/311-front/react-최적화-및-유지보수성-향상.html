React 최적화 및 유지보수성 향상

개요

AI가 발전하면서 단순 react코드를 어느정도 잘 짜게 됐다. 단순히 화면은 이미 완벽하다고 본다.
하지만 컨벤션 없는 코드는 나중에 유지보수, 최적화에 매우 불리하다.
유지보수/최적화에 좋은 코드를 짜기 위한 내용들에 대해서 확인해보자

기본내용

  • 상태 기본값을 설정할 것 => 이후 해당 상태를 사용하는 곳의 타입검증이 필요없어짐
  • 무거운 연산은 useMemo => 웬만하면 비싼 것만 할 것.
  • 컴포넌트 내에 컴포넌트 만들지 말것 => 컴포넌트끼리 결합도가 강해짐
  • arrow function displayname 사용(특히 HOC) => 디버깅에 용이

추가내용

  • 컴포넌트 내 상태에만 종속된 상태는 단순변수처리 => 리렌더링 감소
  • props를 객체로 넘기는 것을 지양하기. => 리렌더링 유발됨
  • props로 내리기 전에 한번 발라내기 ...props 주의하기 => 유지보수 증가
  • useEffect 안쪽에 기명함수 넣기 => 디버깅 용이
  • useEffect는 한가지 역할만 할 것 => 과한 dependencies를 줄이고, 기명함수 작성으로 하나의 일만 하는지 확인

초기값 세팅으로 복잡한 타입체크를 제외할 수 있음.

리렌더링시 자동으로 변경되는 단순변수처리

어차피 state, props가 변하면 리렌더링 될 건데, 굳이 useEffect를 쓸 필요가 없음.

function RefInsteadOfState(){

복잡할수록 단순하게(KEEP IT SIMPLE STUPID)

복잡한 상태들은 하나로 통합해서 사용한다.

  1. 하나의 변수로 관리
  2. 상태를 객체로 관리 : N개의 상태를 1개의 객체로 사용
  3. useReducer 사용

const INIT_STATE={
		init:true,
		loading:false,
		finish:false,
		error:false
}

const stateReducer = (state,action) =>{
	switch (action.type){
		case 'FETCH_LOADING':
			return {isLoading:true, isSuccess:false, isFail:false }
		case 'FETCH_SUCCESS':
			return {isLoading:true, isSuccess:false, isFail:false }
		case 'FETCH_FAIL':
			return {isLoading:true, isSuccess:false, isFail:false }
		default:
			return INIT_STATE;
			break;
	}
}

	if(state.error) return <errorComp/>
}

컴포넌트 구성하기

컴포넌트를 구성할 때 어떻게 구성할지 CONVENTION이 있다면 나중에 유지보수 할 때 편하게 할 수 있고, 예측하지 못한 오류를 잡을 수 있다.

// 1. import 문
import { useState, useEffect, useRef } from 'react'
import { useCustomHook } from './hooks'

// 2. 상수 정의
const DEFAULT_COUNT = 0
const MAX_COUNT = 100

// 3. 타입 정의
interface Props {
  initialCount?: number
  onCountChange?: (count: number) => void
}

// 4. 컴포넌트 정의
const CounterComponent = ({ initialCount = DEFAULT_COUNT, onCountChange }: Props) => {
  // 4-1. Ref
  const inputRef = useRef<HTMLInputElement>(null)
  
  // 4-2. Context, Router
  const theme = useTheme()
  const navigate = useNavigate()
  
  // 4-3. Custom Hooks
  const { user } = useAuth()
  
  // 4-4. useState
  const [count, setCount] = useState(initialCount)
  const [error, setError] = useState(null)
  
  // 4-5. Derived State (계산된 값)
  const isMaxReached = count >= MAX_COUNT
  
  // 4-6. Event Handlers
  const handleIncrement = () => {
    if (count < MAX_COUNT) {
      setCount(prev => prev + 1)
      onCountChange?.(count + 1)
    }
  }
  
  const handleDecrement = () => {
    setCount(prev => Math.max(0, prev - 1))
    onCountChange?.(count - 1)
  }
  
  // 4-7. useEffect
  useEffect(function logCountChange() {
    console.log('Count changed:', count)
  }, [count])
  
  useEffect(function focusInput() {
    inputRef.current?.focus()
  }, [])
  
  // 4-8. Render
  return (
    <div>
      <input ref={inputRef} value={count} readOnly />
      <button onClick={handleDecrement}>-</button>
      <button onClick={handleIncrement} disabled={isMaxReached}>+</button>
      {isMaxReached && <p>Maximum reached!</p>}
    </div>
  )
}

export default CounterComponent

useEffect 안쪽에 기명함수 넣기

내부에 기명함수를 넣으면 callstack에서 해당 함수가 쌓임.

useEffect(function addEvent(){
	document.addEventListener();

	return function removerEvent(){
		document.removeEventListener();
	}
}, [])

Arrow Function displayName사용하기

// 나쁜 예
const withAuth = (Component) => {
  return (props) => { // DevTools에서 <Component>로 표시됨
    const isAuth = useAuth()
    return isAuth ? <Component {...props} /> : <Login />
  }
}

// 좋은 예
const withAuth = (Component) => {
  const WrappedComponent = (props) => {
    const isAuth = useAuth()
    return isAuth ? <Component {...props} /> : <Login />
  }
  
  WrappedComponent.displayName = `withAuth(${Component.displayName || Component.name})`
  return WrappedComponent
}

// 더 좋은 예: 함수 선언식 사용
const withAuth = (Component) => {
  function WithAuth(props) { // DevTools에서 <WithAuth>로 표시됨
    const isAuth = useAuth()
    return isAuth ? <Component {...props} /> : <Login />
  }
  
  WithAuth.displayName = `withAuth(${Component.displayName || Component.name})`
  return WithAuth
}
